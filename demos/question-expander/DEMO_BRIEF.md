# Question Expander Demo Brief

## 这是什么

这是一个把用户输入的一句话，先拆成多条可继续追问的思考路径的 DEMO。

它不急着直接回答，而是先帮助用户看到：

- 可以从哪些角度继续想
- 哪些地方还没有想透
- 下一步更值得追问什么

## 一句话演示话术

你可以把它理解成一个“先展开、再判断”的思考引擎。

传统问答更像直接给答案；这个 DEMO 更像先把问题拆开，让人看到后面真正值得深入的路径。

## 演示时怎么讲

### 开场

这个 DEMO 展示的不是“回答问题”，而是“把问题展开”。

当用户输入一句模糊的问题时，系统会先生成几条不同的思考路径。每条路径都不是结论，而是一个还值得继续推进的方向。

### 用户能看到什么

1. 输入一句话，比如“我想提高工作效率”
2. 系统展开出多条不同路径
3. 每条路径会显示：
   - 这条路径在讲什么
   - 当前还没想透的地方
   - 下一步最值得继续追问的问题
4. 用户可以继续展开到更深层
5. 用户也可以点“停一下”，查看阶段性总结
6. 用户可以复制当前展开结果为 Markdown，沉淀到笔记或文档里

### 这个 DEMO 的亮点

- 不是单轮答案，而是树状思考过程
- 支持多层递进展开
- 深层阅读时可以聚焦当前分支
- 复制结果时会保留已展开结构和“停一下”内容
- 可以接 replay，也可以接 live 大模型

## 推荐演示流程

1. 输入一个开放问题
2. 点击 `Expand`
3. 先介绍第一层的 3 条不同方向
4. 选择其中一条继续展开到 2 到 4 层
5. 在其中一个节点点一次 `停一下`
6. 再继续展开到更深层，展示“聚焦当前分支”
7. 最后点一次 `复制 Markdown`，说明结果可以沉淀到其他笔记系统

## 启动方式

### 1. 启动 adapter

推荐稳定演示用 replay：

```bash
npm run adapter:replay
```

如果要接 live 大模型：

```bash
cp .env.live.example .env.live.local
```

填入你自己的：

- `HWP_LLM_BASE_URL`
- `HWP_LLM_MODEL`
- `HWP_LLM_API_KEY`

然后启动：

```bash
npm run adapter:live:file
```

### 2. 启动前端

```bash
npm run dev
```

默认访问：

```text
http://localhost:5173
```

## 当前适合怎么用

### 最适合

- 内部演示
- 产品方向讨论
- 对外展示概念原型
- 验证“问题展开”交互方式

### 现在已经具备

- 多层展开稳定
- 深层路径不会错乱
- replay 模式可稳定演示
- live 模式可接兼容 OpenAI 的大模型
- 复制结果支持 Markdown

## 当前边界

### replay 模式

- 最稳定
- 最适合演示
- 结果不会随输入实时变化

### live 模式

- 可以实时生成
- 需要你自己的模型配置
- 最好在正式演示前先做一次实测

## 演示时建议避免的误解

- 这不是一个“直接给答案”的问答机器人
- 这更像一个“帮你把问题继续展开”的思考界面
- replay 结果是固定回放，不代表每次输入都会现场推理

## 建议收尾说法

这个 DEMO 目前已经能完整展示核心交互：从一句输入出发，展开出多条可递进的思考路径，并把过程沉淀为结构化结果。

如果下一步继续产品化，重点就不再是“能不能跑”，而是：

- live 结果质量
- 不同行业场景模板
- 更完整的记录和协作能力
